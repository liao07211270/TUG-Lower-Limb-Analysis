# 退化性膝關節炎患者下肢動作分析

## 專題簡介
本專案旨在解決臨床上 TUG（Timed Up and Go）高度仰賴人工計時的痛點。  
傳統 TUG 測試僅能提供總完成時間單一指標，對於退化性膝關節炎（KOA）患者
來說，無法精確呈現個別動作的時長，難以客觀反映病程進展。

本專題以 OPAL IMU 穿戴式感測器為核心，結合機器學習訓練，目標打造一套能
自動解析各動作階段並精準量化時間的評估系統，將傳統測試轉化為具備高客觀性
與臨床價值的數位化輔助工具。

## 實驗設計

### 受試者與感測器配置
- 7 位健康受試者
- OPAL IMU 感測器配戴位置：腰部、雙側大腿、雙側腿骨
- 取樣頻率：128 Hz

### TUG 測試流程
站起 → 直線行走 3 公尺 → 轉身 → 返回 → 坐下

### 動作標記
使用 Kinovea 影像輔助分析工具，逐幀標記 7 個動作區間，建立 Ground Truth

---

## 系統架構

### 資料前處理
- 計算二軸與三軸合成值，消除方向偏誤干擾
- 採用 Sliding Window 技術進行訊號切割
- 固定重疊率（SP）：60 points
- 測試視窗大小：`32`、`48`、`64`、`96`

### 特徵提取
每個視窗提取以下 8 項統計特徵：

| 特徵 | 說明 |
|------|------|
| Mean | 平均值 |
| Std | 標準差 |
| Max | 最大值 |
| Min | 最小值 |
| Range | 全距 |
| Var | 變異數 |
| Skewness | 偏態 |
| Kurtosis | 峰態 |

### 時序修正演算法
依據人體物理運動規律，自動修正不符合邏輯的動作跳轉，優化最終辨識結果

---

## 機器學習模型

### 交叉驗證方式
Leave-One-Subject-Out（7-fold），以受試者為單位分組

### 比較模型
- Decision Tree
- Random Forest
- AdaBoost

### 分類目標
將站起、行走、轉彎三大核心動作做為分類目標（去程與回程合併處理）
